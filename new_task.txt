Необходимо реализовать систему обработки заказов, где создание заказа включает несколько шагов, каждый из которых может завершиться успешно или с ошибкой. Saga Pattern будет использоваться для обеспечения согласованности данных при распределенных транзакциях.

Требования
Бизнес-процесс:
Создание заказа → Проверка наличия товара → Резервирование товара → Создание платежа → Подтверждение заказа
Каждый шаг выполняется отдельным сервисом
При ошибке на любом шаге должны выполняться компенсирующие действия

Технологии:
Java 17+
Spring Boot
Spring State Machine или собственный движок Saga
Kafka/RabbitMQ для межсервисного взаимодействия
База данных (PostgreSQL/MongoDB)
Docker для контейнеризации

Дополнительно:
Логирование всех этапов Saga
Возможность ручного запуска компенсирующих действий
Визуализация состояния Saga
